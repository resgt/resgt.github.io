1.MybatisPlus

MyBatis-Plus是一个开源的MyBatis增强工具包，它简化了在Java应用中使用MyBatis的开发流程，提供了更为便捷的CRUD操作，以及强大的代码生成功能。

MyBatis-Plus的主要特点包括：

  1. 简化开发：提供了大量的通用接口和实现，使得开发者无需手写大量的重复代码，如Mapper接口、Service接口等。
  2. 支持自动代码生成：可以根据数据库表生成实体类、Mapper接口、Service接口等代码，提高了开发效率。
  3. 支持分页查询：提供了基于物理分页和基于内存分页两种分页方式，支持动态排序。
  4. MyBatis-Plus的使用非常简单，只需要在项目中引入相关依赖，配置好数据库连接信息和实体类映射关系即可开始进行CRUD操作。

2.前后端分离概念

前后端分离是一种软件架构模式，将前端和后端开发分离为两个独立的应用程序。前端应用程序负责展示界面和交互逻辑，后端应用程序则负责处理业务逻辑和数据存储等任务。

在传统的Web应用程序中，前端和后端是耦合在一起的，前端页面由后端渲染，前端与后端的交互也由后端控制。而在前后端分离的架构中，前端和后端通过API进行通信，前端通过API获取数据并展示页面，用户与前端进行交互，前端再通过API将用户的操作请求发送到后端进行处理。

前后端分离的优点包括：

  1. 前后端各自独立开发，可以提高开发效率和代码质量。
  2. 前端与后端分离，可以让前端更加专注于界面设计和用户体验，后端更加专注于业务逻辑和数据存储。
  3. 通过API进行通信，可以提高系统的扩展性和可维护性。
  4. 前后端可以选择不同的技术栈进行开发，可以更好地满足项目需求。

前后端分离的缺点包括：

  1. 由于前后端需要通过API进行通信，因此在性能和安全方面需要更多的考虑。
  2. 前后端分离需要更多的技术和人力资源，需要专业的前端和后端开发团队。
  3. 前后端分离的代码结构相对复杂，需要更好的项目管理和协作机制。

3.swagger 接口测试工具

Swagger是一个开源的API开发工具，提供了一系列工具来创建、文档化、测试和部署API。Swagger UI提供了一个可视化的接口测试工具，可以方便地测试API的各种接口请求，并在页面上直接查看接口的响应结果。使用Swagger进行接口测试可以极大地提高开发效率和测试效果。在Swagger UI中，我们可以通过简单的界面操作来构建和发送HTTP请求，然后查看响应结果、HTTP状态码等详细信息，这样能够大大提高我们对API的理解和调试效率。

7.nginx

Nginx是一个高性能的开源Web服务器，在负载均衡的应用场景中，Nginx 作为反向代理服务器，可以接受客户端的请求，并将这些请求转发到后端的多个应用服务器上，从而分摊服务器的压力，提高网站的访问速度和并发量。Nginx 作为负载均衡器的主要作用就是将请求均匀地分发到不同的服务器上。通过编辑Nginx配置文件添加负载均衡器名称并添加反向代理将请求发送到负载均衡器实现。

10.SDK

SDK是Software Development Kit的缩写，翻译为“软件开发工具包”。它是为特定软件开发平台或硬件设备提供的一套软件开发工具、API文档、示例代码等资源的集合，通常包括编译器、调试器、库文件、文档等工具，开发者可以利用这些工具来开发相应的软件或应用程序。SDK的存在可以简化软件开发流程，提高开发效率和开发质量。

11.微服务概念

微服务是一种软件开发方法，它将应用程序划分为小型、独立的服务，每个服务都具有自己的独立进程和数据存储。这些服务可以在不同的语言、平台和技术堆栈之间进行通信，通常使用HTTP或消息传递协议。

微服务产生的原因主要是应对传统单体应用程序的缺点。单体应用程序随着规模的增长变得越来越难以维护和扩展，因为它们通常是一个巨大的、紧密耦合的代码库。微服务通过将应用程序拆分成小型服务，每个服务都专注于一个具体的功能，从而使应用程序更加灵活、可扩展和易于维护。

21.maven

Maven是一个基于Java的自动化构建工具，它提供了一个能够自动化构建项目、管理依赖、运行测试以及发布产物的框架。使用Maven可以让项目构建和依赖管理更加方便，节省时间和精力。它使用pom.xml文件来描述项目依赖和配置信息，并且可以通过命令行或者IDE插件来执行构建任务。Maven的核心理念是约定优于配置，它提供了一系列默认配置和约定，可以极大地简化构建配置的复杂性。
