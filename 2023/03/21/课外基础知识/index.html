拷贝？深拷贝与浅拷贝？在c++中如何实现

浅拷贝只是复制了指针或引用，而没有复制对象本身。这意味着两个对象将共享同一个内存空间，对一个对象的修改也会影响另一个对象。这种拷贝方式通常用于简单的数据类型或者仅包含指针或引用的对象。

深拷贝会创建一个新的对象，然后将原始对象的值复制到新对象中。这意味着两个对象有不同的内存空间，修改一个对象不会影响另一个对象。这种拷贝方式通常用于包含指针或引用的复杂数据类型。

在c++中浅拷贝使用等号赋值，深拷贝使用memcpy

常用排序方法

冒泡排序（Bubble Sort）

冒泡排序是一种简单的排序算法。它重复地遍历待排序的元素列表，比较相邻的两个元素，并交换它们的位置，直到整个列表排序完成。冒泡排序的时间复杂度为 O(n^2)。

选择排序（Selection Sort）

选择排序是一种简单的排序算法。它重复地找到列表中最小的元素，并将其放在已排序的元素的末尾。选择排序的时间复杂度为 O(n^2)。

插入排序（Insertion Sort）

插入排序是一种简单的排序算法。从未排序序列中选一个插入到已排序序列的合适位置，然后重复这个过程，直到整个列表排序完成。插入排序的时间复杂度为 O(n^2)。

快速排序（Quick Sort）

快速排序是一种常用的排序算法。它通过不断地分割列表来排序。首先，选择一个基准元素，然后将列表中小于基准元素的元素放到左边，大于基准元素的元素放到右边。然后，对左右两个列表分别递归进行快速排序，最终将它们合并起来。快速排序的时间复杂度为 O(nlogn)。

归并排序（Merge Sort）

归并排序是一种分治排序算法。它将列表分成两个子列表，然后对每个子列表进行递归排序，最后将两个有序的子列表合并起来。归并排序的时间复杂度为 O(nlogn)。

堆排序（Heap Sort）

堆排序是一种基于堆的排序算法。它将列表看作一棵完全二叉树，并将它转换成一个最大堆。然后，将堆的根节点移动到已排序的元素列表的末尾，并重新构建最大堆。重复这个过程，直到整个列表排序完成。堆排序的时间复杂度为 O(nlogn)。

计数排序（Counting Sort）

计数排序是一种基于计数的排序算法。它先计算每个元素在列表中出现的次数，然后根据这些计数将元素排序。计数排序的时间复杂度为 O(n+k)，其中 k 是元素的最大值。

联合体和结构体

结构体和联合体都是C语言中用来组织数据的复合数据类型，不同之处在于：

  • 结构体中的成员变量占用不同的内存空间，可以同时访问结构体中的多个成员变量。
  • 联合体中的所有成员变量共用同一块内存空间，同一时间只能使用联合体中的一个成员变量，对其他成员变量的修改会影响到共用同一内存空间的其他成员变量。

数组和指针的区别

  1. 类型不同：数组是一组相同类型的数据元素的集合，而指针是一个指向某种类型数据的地址值。
  2. 内存分配方式不同：数组在定义时就要指定元素个数，其内存空间是连续分配的，而指针是在运行时动态分配的，其内存空间不一定是连续的。
  3. 大小不同：数组的大小在定义时就已经确定，而指针的大小在不同的操作系统和编译器中可能不同。
  4. 初始化方式不同：数组可以通过在定义时初始化所有元素，也可以通过逐个赋值来初始化。指针则需要指向一个已经存在的变量或分配的内存块。
  5. 使用方式不同：数组可以用于存储一组相同类型的数据，而指针可以用于指向单个变量或一块内存空间，也可以用于函数参数的传递等。

总的来说，数组和指针在某些情况下可以互相替代，但它们的本质是不同的。数组是一种静态的数据结构，而指针是一种动态的数据结构。在程序设计中，需要根据实际需求选择合适的数据类型。

函数调用

函数调用是程序中常用的一种机制，它用来执行已定义好的函数并返回结果。函数调用的实现可以分为以下几个步骤：

  1. 保存现场：在函数调用前，需要先保存当前函数执行的现场信息，包括函数返回地址、函数参数、局部变量和寄存器等。
  2. 跳转到函数：将控制权转移到被调用函数的代码段，并将函数参数传递给被调用函数。
  3. 执行函数：被调用函数根据传递的参数执行相应的操作，可以访问传递进来的参数、局部变量和全局变量等。
  4. 返回结果：函数执行完毕后，将结果返回给调用者，并恢复之前保存的现场信息。
  5. 恢复现场：将之前保存的现场信息恢复，包括恢复返回地址、恢复局部变量和寄存器等，回到调用者的代码段继续执行。

这些步骤通常是由编译器或者解释器自动完成的，但也可以手动实现。在不同的计算机系统中，函数调用的实现方式可能会有所不同，但基本原理是相同的。

动态查找有哪些

动态查找（Dynamic Search）是指在一个动态的数据集合中，根据一些特定的要求（例如某个关键字）查找满足条件的数据记录。与之相对的是静态查找，静态查找是在一个静态的数据集合中进行查找。

常见的动态查找算法包括：

  1. 二叉查找树（Binary Search Tree，BST）：通过构建一棵二叉树，在每个节点中存储一个关键字，实现快速的查找和插入操作。
  2. 平衡二叉查找树（Balanced Binary Search Tree）：为了避免BST在某些极端情况下退化成链表的问题，产生了一些平衡二叉查找树的变种，如红黑树（Red-Black Tree）、AVL树等。
  3. 哈希表（Hash Table）：将关键字通过哈希函数映射到一个固定大小的数组中，实现快速的查找和插入操作。
  4. B树（B-Tree）：多路平衡查找树，能够在一个节点中存储多个关键字，有效地减少了树的高度。

常见八股

什么是C++？

C++是一种面向对象的高级编程语言，是C语言的扩展。它提供了更多的特性，如封装、继承、多态等。

什么是指针？

指针是一个变量，它存储了一个内存地址。通过指针，可以访问该地址处的变量或对象。

什么是引用？

引用是一个别名，它指向另一个变量或对象。通过引用，可以使用原始变量或对象的别名来操作它们。

什么是函数重载？

函数重载是指定义多个具有相同名称但参数列表不同的函数。编译器会根据传递给函数的参数类型和数量来确定应该调用哪个函数。

什么是虚函数？

虚函数是在基类中声明的一种特殊的函数。当在派生类中重写该函数时，可以使用虚函数的特性来实现多态。

什么是构造函数和析构函数？

构造函数是一种特殊的函数，用于在创建对象时初始化对象的成员变量。析构函数用于在对象被销毁时清理对象的资源。

什么是类？

类是一种用户定义的数据类型，它封装了数据和操作数据的函数。它是面向对象编程的基本单位。

什么是继承？

继承是一种面向对象的特性，它允许一个类从另一个类继承数据和函数。派生类可以重写基类的函数，从而实现多态。

什么是模板？

模板是一种通用的编程机制，它允许编写通用的代码来处理不同类型的数据。通过模板，可以在编译时生成具体的代码。

什么是STL？

STL是标准模板库，它是C++的一个库，提供了许多通用的数据结构和算法。STL包括容器、迭代器、算法和函数对象等。

逻辑结构和物理结构

逻辑结构是指数据之间的逻辑关系，如线性结构、树形结构、图形结构等。这种结构与具体的物理存储方式无关，只涉及到数据元素之间的关系和相互作用。

物理结构是指数据在计算机内部的存储方式，包括顺序存储结构和链式存储结构两种。顺序存储结构是指数据元素按照逻辑次序存储在一段连续的存储空间中，而链式存储结构则是通过指针将数据元素存储在不同的物理地址中，并通过指针来描述元素之间的逻辑关系。

数据库基本知识

  1. 数据库是什么？ 数据库是一种可以存储和管理大量数据的系统。它通常由一个或多个数据表组成，每个数据表由多个数据行和数据列组成，每个数据行表示一个数据记录，每个数据列表示数据的一个属性。
  2. 数据库管理系统（DBMS） 数据库管理系统是用于管理数据库的软件，可以对数据库进行访问、查询、修改和管理等操作。常见的DBMS包括MySQL、Oracle、SQL Server等。
  3. SQL SQL（Structured Query Language）是一种用于管理和操作关系型数据库的标准语言。它可以用来进行数据查询、插入、更新和删除等操作。
  4. 关系型数据库 关系型数据库是一种基于关系模型的数据库，其中数据以表格形式存储，并使用关系代数等理论来管理数据之间的关系。关系型数据库的优点是数据结构清晰、操作简单、支持事务等。
  5. 非关系型数据库 非关系型数据库（NoSQL）是指不使用关系模型的数据库，它们通常使用键值对、文档、列族等方式来存储数据，比如MongoDB、Redis等。非关系型数据库的优点是可扩展性强、适用于分布式环境等。
  6. ACID属性 ACID是指数据库事务应该具备的四个属性，即原子性（Atomicity）、一致性（Consistency）、隔离性（Isolation）、持久性（Durability）。这些属性保证了数据库在处理事务时的可靠性和正确性。
  7. 数据库索引 数据库索引是用于加速数据库查询操作的一种数据结构，通常由一个或多个字段构成，可以提高查询效率和数据检索速度。

数据库范式

数据库范式是关系数据库中用于规范化数据结构的一组规则，目的是提高数据库的数据存储效率和查询效率，并避免数据冗余和错误。常见的数据库范式包括第一范式（1NF）、第二范式（2NF）、第三范式（3NF）等。

  1. 第一范式（1NF） 第一范式要求数据表中每个字段都应该是原子性的，即不能再分解为更小的数据项。这样可以避免数据冗余和数据错误。
  2. 第二范式（2NF） 第二范式要求一个数据表中应该只描述一个事物，而不是多个事物。同时，如果数据表中存在非主键字段与主键只有部分依赖的情况，应该将这些非主键字段分解到另外一个数据表中，以保证数据表中每个字段都和主键有完全依赖关系。
  3. 第三范式（3NF） 第三范式要求数据表中不存在非主键字段与主键之间的传递依赖关系。如果存在这种依赖关系，应该将这些非主键字段分解到另外一个数据表中，以保证数据表中每个字段都和主键有直接依赖关系。

在实际的数据库设计过程中，可以根据需求和数据结构的复杂程度，选择不同的范式来规范化数据库结构。在范式化过程中，需要权衡数据库的性能和数据的一致性和可靠性。

范式化的优点包括：

  • 减少数据冗余，节省存储空间
  • 数据库更新和修改时，不需要修改多个表，提高数据一致性和可靠性
  • 数据查询效率更高，提高数据库性能

但范式化的缺点也是显而易见的，包括：

  • 数据表结构复杂，难以维护
  • 数据查询需要多表连接，降低数据库查询性能
  • 范式化可能会忽略某些业务场景的特殊需求，导致查询效率低下

因此，在实际的数据库设计中，需要权衡不同的需求和实际情况，选择适合的范式化方式。

指向整型变量的指针和指向结构体的指针哪个占用空间更大

指向整型变量的指针和指向结构体的指针在内存中占用的空间大小是相同的，都是根据计算机体系结构的位数而定。在大多数计算机体系结构中，一个指针占用的空间大小是4字节或8字节，取决于操作系统和CPU的位数。
