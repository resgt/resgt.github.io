进程与线程的区别

进程是资源分配的基本单位，每个进程都有自己独立的内存空间，内存中至少有一个进程，一个进程可以包含多个线程

线程是资源分配的基本单位，一个进程内的线程共享进程的存储空间，

（5）进程的状态转换

就绪 分配到出处理机之外的所有资源

执行 进程获得处理机其程序正在处理机上执行

阻塞 正在执行的程序由于请求申请某种资源而无法继续执行 便会让出处理及进入阻塞态，待申请的资源就位后会进入就绪态

（5）死锁

概念：死锁是指多个进程因竞争资源而造成的一种僵局，若无外力作用这些进程都将无法向前推进

四个必要条件：互斥 不剥夺 循环等待 请求和保持

哲学家就餐问题：解决方法 限制最多就餐人数、限制哲学家只拿左手或右手、保证左右手都能拿再拿

银行家算法：最具代表性的死锁避免算法 需要获取进程运行的所有资源信息、不会给可能造成死锁的进程分配资源

四个数据结构：可用资源、最大需求、分配矩阵、需求矩阵

（4）内存管理

页是信息的物理单位，分页是为了实现非连续分配，满足系统管理的需要。段是信息的逻辑单位，分段的目的是为了更好的实现共享，满足用户的需要。

页的大小固定，每个进程都有一个页表。段的长度不固定，用户根据自己的程序设置端的长度

页式存储可能产生内部碎片 段式存储可能产生外部碎片

分页的地址空间是一维的，分段的地址空间是二维的

虚拟内存 共享内存

虚拟内存是计算机系统内存管理的一种，它使得应用程序认为他可以拥有连续可用的内存，但实际上被分割为多个物理内存碎片，部分不常用的块会被暂时存储到外存中在需要的时候调入。

共享内存式进程间通信机制，操作系统在几个进程的地址空间上映射一段内存然后几个进程可以在不需要调用系统函数的情况下在那段内存进行读写操作

改进形时钟 考虑访问和修改位，优先淘汰未访问的

（3）文件系统

有结构文件

文件系统是操作系统中用于管理文件和目录的机制。它可以让用户在计算机上创建、存储、组织和查找文件。

FAT是一种基于表格的文件系统，使用一个称为FAT表格的结构来跟踪每个文件的存储位置。每个FAT表项指向存储文件数据的磁盘扇区。由于FAT文件系统不需要存储大量元数据，因此通常被用于存储小容量的移动设备上。FAT文件系统有多个版本，其中最常见的是FAT16、FAT32和exFAT。

Inode是Unix/Linux文件系统中的一种数据结构，用于存储文件和目录的元数据。每个文件或目录在文件系统中都有一个唯一的inode号码。inode记录了文件的所有者、文件权限、文件大小、文件类型等信息。它还存储了文件的物理位置信息，包括数据块的位置、大小和使用情况等。由于Inode记录了更多的元数据，因此相对于FAT文件系统，它需要更多的存储空间来管理文件系统。

（4）存储器

RAM 和 ROM

RAM 随机存取存储器时与CPU直接交换数据的内部存储器，可以随时读写并且速度很快。断电后数据消失

ROM只读存储器 速度较慢 但数据稳定 断电后数据不会改变

SRAM具有静态存取功能的内存 不需要刷新电路 速度快集成度低 功耗大

DRAM 只将数据保存很短时间 必须隔一段时间刷新一次。

译码阶段：解析IR中的指令 具体要操作哪些寄存器数据或内存地址

执行阶段 执行指令完成数据传输或地址跳转

中断 检测是否有中断请求

（3.5）数据库范式

数据库范式是一种设计数据库表格的方法，可以让我们更好地组织数据。

一范式 ：属性不可分。即数据库表中的每一列都是不可分割的基本数据项，同一列中不能有多个值，即实体中的某个属性不能有多个值或不能有重复的属性

二范式：要求表格中的每个非主键列都完全依赖于主键，消除了部份依赖。这意味着，如果我们有一个包含学生信息和成绩的表格，学生ID应该是主键，成绩不应该被分割成不同的列。

三范式：非主属性不传递函数依赖于键码 简而言之 第三范式就是树形不依赖于其他非主属性

要求表格中的每个非主键列都不依赖于其它非主键列。这意味着，如果我们有一个包含订单信息的表格，订单号应该是主键，顾客的姓名和地址应该单独存在另外一个表格中，而不是和订单信息一起存在一个表格中。

(3.5)数据库类型

  1. 关系型数据库：以表格形式存储数据，并通过表格之间的关系来组织数据。常见的关系型数据库包括 MySQL、Oracle、SQL Server 等。
  2. 非关系型数据库：以键值对的形式存储数据，不需要预先定义表结构。常见的非关系型数据库包括 MongoDB、Redis、Cassandra 等。
  3. 图形数据库：以图形的形式存储数据，并通过节点之间的关系来组织数据。常见的图形数据库包括 Neo4j、OrientDB 等。
  4. 文档数据库：以文档的形式存储数据，文档可以是 JSON、XML 等格式。常见的文档数据库包括 MongoDB、CouchDB 等。

（3.5）关系型数据库特点

  1. 数据以表格的形式进行存储，表格中的每一行都代表一个记录，每一列代表一个属性。
  2. 数据之间存在关系，可以通过关系进行联结查询。
  3. 支持 SQL 查询语言，可以通过 SQL 语言对数据库进行操作。
  4. 支持事务，可以确保数据的完整性和一致性。
  5. 可以对数据进行约束和索引，提高数据的查询效率。
  6. 适合处理结构化数据，如订单、用户信息、日志等。

（2.5）什么是事务 ？什么是锁？事物的四个特性？

(需改进)什么是事务？

​ 事务是指作为单个逻辑⼯作单元执⾏的⼀系列操作，要么完全地执⾏，要么完全地不执⾏。 事 务处理可以确保除⾮事务性单元内的所有操作都成功完成，否则不会永久更新⾯向数据的资源。通 过将⼀组相关操作组合为⼀个要么全部成功要么全部失败的单元，可以简化错误恢复并使应⽤程序 更加可靠。

什么是锁？

​ 锁是⽤于解决隔离性的⼀种机制。事务的隔离级别通过锁的机制来实现。

四个特性：

原子性：全做或全不做

一致性：事务开始和结束后完整性未被破坏

隔离性： 只允许一个事务请求同一数据 不同事物之间没有干扰

持久性：事务完成后 更新保存到数据库不能回滚

（2.5）软件工程

黑盒测试：将程序看作一个不能打开的黑盒子 只关注程序能否根据输入得到正确的输出

白盒测试：旨在测试软件代码的内部结构、逻辑和工作方式，以验证其是否符合设计和规范要求。

区别

测试方式不同 ：功能测试 结构性测试

测试目的不同：将程序看作黑盒子 通过测试内部结构逻辑验证是否符合设计要求

测试原则不同

（2）开发模型

瀑布模型：一种顺序执行的开发模型，以阶段性为基础，需求分析、设计、编码、测试、维护等一系列步骤依次执行。

敏捷开发：一种基于迭代、协作和自适应的开发模型，强调快速响应需求变化和不断迭代优化，以最终交付高质量的软件为目标。

增量模型：一种逐步增量式的开发模型，每个增量都是完整的软件系统，系统的功能逐步增加，开发过程中需要不断重构和修改系统设计。

螺旋模型：一种循环迭代的开发模型，每个迭代都包括风险分析、需求分析、设计、开发、测试和评估等步骤，强调风险管理和反馈。

（2）设计模式

  1. 单例模式：确保一个类只有一个实例，并提供全局访问点。在一个项目中，有些对象需要被全局使用，此时就可以采用单例模式，比如日志系统。
  2. 工厂模式：将创建对象的过程封装起来，通过调用工厂方法来获取对象，从而避免了直接创建对象。在一个项目中，有些对象的创建过程很复杂，需要封装起来，比如网络请求的工厂类。
  3. 观察者模式：定义了一种一对多的关系，使得多个观察者对象同时监听某一个主题对象，当主题对象状态发生改变时，它的所有依赖者（观察者）都会收到通知并自动更新。在一个项目中，当一个对象的改变需要通知其他对象时，可以采用观察者模式，比如一个订单状态变化需要通知多个观察者。

（3.5）介绍下MVC模式

MVC模式是软件工程中的一种软件架构模式，他把软件分为三个基本部分 模型、视图、控制器

其中 模型由程序员编写应有的功能 数据库专家进行数据管理和设计

视图 界面设计人员进行图形界面设计

控制器 负责转发请求 对请求进行处理
